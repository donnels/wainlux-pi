= ADR-019: Distributed Architecture Capability
:revdate: 2026-02-13

== Status

ACCEPTED (partly implemented)

== Context

Pi Zero W (512MB RAM, ARMv6 CPU) runs Flask + K6 driver.
MCP dependency stack is hard to build reliably on this hardware.
UI split may still be needed later.

== Decision

Design and run in distributed mode where needed.
Base64 image embedding stays (no filesystem dependencies).
MCP runs outside Pi in its own container on modern 64-bit hosts.
Pi is API + driver by default.

== Rationale

*Portability:* Jobs are self-contained JSON. Save/load works anywhere.

*Distribution ready:* UI can move to desktop. Pi becomes API-only. Same job format.

*No premature optimization:* Monolith works now. Split later if needed.

*Migration path:* Add `/api/engrave/from-job` endpoint when required. Accept complete job JSON with embedded base64. No data model changes.

*MCP decoupling:* Keep MCP off Pi now. Keep it containerized. Bring it back to Pi only with better hardware.

== Consequences

=== Positive

* Portable jobs (base64 embedded)
* Future-proof (can split without breaking)
* Reproducible (no missing files)
* MCP remains available without blocking Pi builds

=== Negative

* Larger payloads (~33% base64 overhead)
* Two-step upload still required (prepare → engrave)
* Extra service/container to operate when MCP is used

=== Neutral

* Frontend split is still an option, not current work
* Review trigger remains: >30s processing time on Pi Zero W
